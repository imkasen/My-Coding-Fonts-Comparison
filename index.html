I've been using Source Code Pro as the coding font for the past few years, but since it doesn't support ligatures, I recently started trying to replace it with Fira Code. I've tried JetBrains Mono and Cascadia Code before, but since I like thin fonts and these two fonts are thicker, I didn't use them as IDE and editor fonts. But for the command-line interface, I choose to use Cascadia Code PL.

中文字体完全一致。Chinese characters are exactly the same.

However, no matter how you choose, there are always some regrets.

Source Code Pro has beautiful fonts and symbols, but unfortunately it does not support ligatures.
Fira Code supports ligatures, but the symbols are not as beautiful as Source Code Pro.
JetBrains Mono is almost great, but there are three things I don't like. One is that its font is slightly thicker than Source Code Pro and Fira Code, the second is that its font looks narrower visually and I prefer a wider font, the third is that I don't like its lowercase letter "u", which is missing a stroke.
Cascadia Code is the thickest of the four fonts, and its "==" symbols are not connected.
The two Chinese fonts are actually identical, but with different names. "HW" means one Chinese character is equal to the width of two English letters.

Update

I found a new font, Iosevka, which is very customizable. I also compared one of its Source Code Pro-like font with the original Source Code Pro. And I am now ready to start using this font as my IDE and editor font.
